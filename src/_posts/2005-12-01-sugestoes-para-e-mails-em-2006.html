Dentre os vários e-mails que lotam nossas caixas-postais, um bom percentual utilizam o recurso de imagens, o chamado formato HTML. Para mim, mensagem eletrônica é texto e a única tabela que existe é a ASCII. Mas concordo que para tudo há uma utilidade, mesmo que bem específicas, como o envio de boletins informativos, por exemplo.

É preciso analisar como estes e-mails serão visualizados, já que filtros (anti-spam) , comum nos clientes de e-mail, bloqueiam as imagens. A mensagem pode se tornar uma conjunto de retângulos, de formas e tamanho variados.

Para reduzir o impacto que os filtros causam, o site CampaignMonitor publicou um conjunto de dicas, vejamos algumas delas:

  • Evitar utilizar imagens para cabeçalhos, botões e links;
  • No topo da mensagem incluir um link: versão web da mensagem;
  • Usar o atribuito alt no código HTML, para identificar melhor cada imagem;
  • Sempre identificar exatamente as dimensões da imagen através dos atributos width e height.
